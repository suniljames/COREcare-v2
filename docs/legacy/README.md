# Legacy reference material

This directory holds **frozen-in-time snapshots of v1 (`COREcare-access`)** for collaborators building v2 without v1 source access. Currently contains one artifact set:

- [`v1-ui-catalog/`](v1-ui-catalog/) — per-persona screenshot catalog of v1's UI surface (issue [#79](https://github.com/suniljames/COREcare-v2/issues/79)).

## What this is

A visual ground-truth reference for v2 contributors. The catalog captures every reachable v1 page per persona at desktop + mobile breakpoints, with structured Markdown captions describing route, persona, viewport, observed CTAs, and observed interactions.

## What this is NOT

- **Not a v2 spec.** v2 is a ground-up rebuild and does not have to look or behave like v1. The catalog informs v2 design decisions but is not normative for them.
- **Not auto-refreshed.** Generated against a pinned v1 commit + frozen seed dataset. Refreshed only on explicit ask.
- **Not customer-facing.** Internal documentation for engineers contributing to v2.

## Sensitivity

**Internal documentation. Generated against PHI-free seed data.** Must not be made public. Must not be uploaded to third-party services (diagram renderers, LLM playgrounds, public gists, transcription tools). Repo-NDA collaborators only.

Every name, address, phone number, DOB, MRN, prescription, and free-text note in the catalog is a placeholder from the locked PHI placeholder set documented in [`../migration/README.md` §PHI Placeholder Convention](../migration/README.md#phi-placeholder-convention). Faker output is **not acceptable** as a substitute (see committee design synthesis on [#79](https://github.com/suniljames/COREcare-v2/issues/79)).

## Audience

v2 contributors, including AI agents acting on their behalf. Particularly: engineers rebuilding the persona-facing frontend who do not have access to the `hcunanan79/COREcare-access` repository.

## Persona vocabulary

The catalog uses the **five locked personas** from [`../migration/README.md` §Personas](../migration/README.md#personas):

| Slug | Canonical name |
|------|----------------|
| `agency-admin` | `Agency Admin` |
| `care-manager` | `Care Manager` |
| `caregiver` | `Caregiver` |
| `client` | `Client` |
| `family-member` | `Family Member` |

Folders use kebab-case slugs; captions and index headings use the canonical capitalized strings. v1 has no platform-operator role per #236; `is_superuser`-gated v1 routes (notably the View-As emergency kill switch) are folded into the Agency Admin catalog under the `100+` numbering convention.

## How to navigate

Start at [`v1-ui-catalog/README.md`](v1-ui-catalog/README.md) — it indexes every captured route, grouped by persona.

Each persona directory holds image files (`<NNN>-<route>.<viewport>.webp`) and sibling caption files (`<NNN>-<route>.md`). The caption file is the cross-reference handle: its frontmatter `canonical_id` field is what the inventory's `screenshot_ref` column points at.

## How to refresh

The catalog is generated by `python manage.py snapshot_ui_catalog` in v1 (`hcunanan79/COREcare-access`). See [hcunanan79/COREcare-access#1489](https://github.com/hcunanan79/COREcare-access/issues/1489) for the management command.

When refreshing:
1. Pull v1 at the desired commit; record the SHA.
2. Load the PHI-scrubbed seed snapshot.
3. Run the management command with `--output-dir <path-to-this-directory>/v1-ui-catalog`.
4. Update this README's pinned references and run [`../../scripts/check-v1-catalog-coverage.sh`](../../scripts/check-v1-catalog-coverage.sh) to confirm parity with the inventory.

## Skip-reason taxonomy

When the inventory's `screenshot_ref` field is `not_screenshotted: <reason>`, `<reason>` should be one of:

| Reason | Meaning |
|--------|---------|
| `pending #79` | Initial catalog work has not landed yet. (Current default.) |
| `destructive_endpoint` | POST/DELETE endpoint that triggers a side effect (send-message, delete-record, dispatch-payment); skipped by crawler. |
| `gated_by_capability` | Route requires a capability the test persona does not hold; coverage gap acknowledged. |
| `no_seed_data` | Route requires a data state the seed snapshot does not provide. |
| `no_authenticated_surface` | Persona has no v1 login route — the row is recorded for completeness but is not screenshottable. |
| `auth_redirect` | Login / magic-link / role-switch routes; intentionally not screenshotted to avoid credential capture. |
| `non_html_response` | Endpoint returns a binary file (PDF / CSV / Excel) via `Content-Disposition: attachment`; nothing to render. |

Authors of new skip-reason values: extend this table in the same PR.
