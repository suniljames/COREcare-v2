**Interaction notes:**
- Endpoint → `client_calendar_api` returns serialized calendar entries with fields `{id, title, start, end, className, type, editable}` for the per-client calendar widget.
- Used by → the FullCalendar-style widget on [agency-admin/052-calendar](052-calendar.md); each view-mode change re-fetches with a new date window.
- Title encoding → includes the unicode caregiver-shift glyph and the caregiver name + scheduled hours; consumers must HTML-escape before rendering.
- "editable" flag → false for shifts (rendered read-only); custom events created via [agency-admin/054-create](054-create.md) come back with `editable: true`.
- Permissions → guarded by per-client view permission; PHI surfaces in the JSON, so consumers should treat the response as untrusted and never mirror it to the family-portal context.
