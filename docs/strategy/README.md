# Strategy

> **Audience:** PMs, founders, business stakeholders.
> **Engineers:** you can ignore this directory. Engineering reference lives in [`../developer/`](../developer/).

Business strategy artifacts — market analysis, competitive landscape, pricing, M&A considerations, MRD, investor materials. These docs are point-in-time analyses (the bracketed numeric prefix is a versioning convention) and may lag current direction; treat anything load-bearing as needing reconfirmation before acting.

## Files

The numeric prefixes group related artifacts:

- `[00.x]` — Market analysis (broad and addenda)
- `[01.0]` — Market analysis (narrow / focused)
- `[02.0]` — Competitive analysis
- `[03.0]` — Investor landscape
- `[04.0]` — M&A analysis
- `[05.0]` — Market Requirements Document (MRD)
- `[06.0]` — Pricing strategy
- `[07.0]` — Press release
- `[08.0]` — External FAQ

If you're an engineer who landed here looking for engineering context, you probably want [`../README.md`](../README.md) (docs map) or the [root `CONTRIBUTING.md`](../../CONTRIBUTING.md).
